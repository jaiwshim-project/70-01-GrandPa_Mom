# /review-evaluate — 검토 및 평가 (Review & Evaluate)

콘텐츠를 **검토(Review)**하여 문제점을 파악하고, 직접 **수정(Fix)**한 뒤, **평가(Evaluate)**하여 품질 점수를 부여한다.
**97점 이상 도달할 때까지 최소 3회 순환**한다.

## Usage
/review-evaluate [file_path_or_directory]

## Phase 1. 평가 대상 결정

### 인자가 있는 경우
- **파일 지정**: 해당 파일을 평가 대상으로 한다.
- **디렉토리 지정**: 디렉토리 내 모든 파일을 재귀 탐색하여 평가 대상으로 한다 (바이너리, 이미지, node_modules 등 제외).

### 인자가 없는 경우
- 직전 `/review-evaluate` 실행 이후에 작업(생성/수정)한 **모든 파일**을 대상으로 한다.
- 대상 파일 목록을 확인하기 위해 git diff, 최근 수정 파일 등을 조회한다.
- 대상이 모호하거나 특정할 수 없는 경우, **AskUserQuestion으로 사용자(PO)에게 평가 대상을 확인**한다.

### 대상 확인 보고
```
📋 평가 대상 확인
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
대상 파일:
  1. {파일 경로}
  2. {파일 경로}
  3. ...

총 {N}개 파일을 검토·평가합니다.
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 다수 파일 평가 방식
- **검토(Review)**: 파일별로 개별 검토하여 문제점 식별
- **수정(Fix)**: 파일별로 개별 수정
- **평가(Evaluate)**: 전체 대상 파일을 **통합하여 하나의 종합 점수** 산정 (5기준 각 20점, 총 100점)
- 순환 루프의 종료 조건(97점)은 통합 종합 점수 기준

## Phase 2. 순환 루프 (Review → Fix → Evaluate)

**최소 3회 반복, 97점 이상 도달 시 종료 조건 진입.**

```
┌─────────────────────────────────────────────┐
│            순환 루프 (Round N)                │
│                                             │
│  Step 1. 검토 (Review)                       │
│    → 문제점 식별 + 심각도 분류                  │
│                                             │
│  Step 2. 수정 (Fix)                          │
│    → 발견된 문제점을 직접 수정                   │
│                                             │
│  Step 3. 평가 (Evaluate)                     │
│    → 5기준 점수 산정                           │
│                                             │
│  판단:                                       │
│    - Round < 3       → 다음 라운드 계속         │
│    - Round >= 3 AND < 97점 → 다음 라운드 계속   │
│    - Round >= 3 AND >= 97점 → Phase 3 진입    │
└─────────────────────────────────────────────┘
```

### Step 1: 검토 (Review)

문제점을 식별하고 심각도를 분류한다.

```
🔍 검토 결과 (Round {N})
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📝 검토 대상: {filename}

🚨 발견된 문제점:

[Critical] — 즉시 수정 필요
• 문제 설명 (파일명:줄번호)
  → 수정 제안

[High] — 우선 수정 권장
• 문제 설명 (파일명:줄번호)
  → 수정 제안

[Medium] — 개선 권장
• 문제 설명 (파일명:줄번호)
  → 수정 제안

[Low] — 선택적 개선
• 문제 설명 (파일명:줄번호)
  → 수정 제안

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 2: 수정 (Fix)

발견된 문제점을 **직접 수정**한다. 제안만 하고 넘어가지 않는다.
- Critical / High: 반드시 수정
- Medium: 가능하면 수정
- Low: 판단하여 선택적 수정
- 해당 심각도의 문제가 없으면 해당 섹션 생략

### Step 3: 평가 (Evaluate)

수정 후 5가지 기준으로 품질 점수를 산정한다 (각 20점 만점, 총 100점).

**5가지 평가 기준:**
- 기술적 정확성 (Technical Accuracy): 정보의 정확성과 최신성
- 가독성 (Readability): 읽기 쉬움과 이해 용이성
- 구조 및 구성 (Structure & Organization): 논리적 흐름과 체계적 배치
- 완성도 (Completeness): 주제의 포괄적 커버리지
- 유용성 (Usefulness): 실용적 활용 가능성

```
📊 평가 결과 (Round {N})
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📝 평가 대상: {filename}
🔄 라운드: {N} / 최소 3회
⭐ 종합 평점: XX/100점

📈 세부 평가:
• 기술적 정확성: XX/20점
• 가독성: XX/20점
• 구조 및 구성: XX/20점
• 완성도: XX/20점
• 유용성: XX/20점

💬 평가 의견:
[강점]
-
-

[개선 필요 사항]
-
-

[이번 라운드 수정 사항]
-
-

➡️ 다음: {다음 라운드 계속 / Phase 3 진입}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Phase 3. 97점 도달 — 종료 확인

97점 이상 도달하고 최소 3회를 충족하면, **사용자(PO)에게 추가 라운드 여부를 확인**한다.

```
✅ 품질 목표 달성 (Round {N})
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⭐ 최종 평점: XX/100점 (목표 97점 달성)
🔄 총 라운드: {N}회

추가 검토·평가를 진행하시겠습니까?
- [완료] 검토·평가 종료
- [추가] 1회 더 검토·평가 수행
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

- **완료** → 최종 결과 보고 후 종료
- **추가** → 1회 더 순환 루프 실행 후 다시 확인

### 최종 결과 보고 형식

"완료" 선택 시 아래 형식으로 최종 보고한다.

```
✅ 최종 검토·평가 결과
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⭐ 최종 평점: XX/100점
🔄 총 라운드: {N}회

📁 평가 대상:
  1. {파일 경로}
  2. {파일 경로}

📝 라운드별 수정 이력:
  Round 1:
  - {수정 내용}
  Round 2:
  - {수정 내용}
  ...

검토·평가 완료.
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Tools Required
- Read, Glob, Grep (검토용)
- Edit (수정용)
- AskUserQuestion (대상 확인 / 종료 확인용)

## Execution Flow 요약

```
1. 평가 대상 결정 (인자 확인 → 없으면 직전 작업 파일 → 모호하면 사용자(PO) 확인)
   - 다수 파일: 파일별 검토·수정, 통합 종합 점수 산정
2. 순환 루프 시작
   Round 1: 검토 → 수정 → 평가
   Round 2: 검토 → 수정 → 평가
   Round 3: 검토 → 수정 → 평가
   ...
   (97점 미만이면 계속 반복)
3. 97점 이상 + 3회 이상 → 사용자(PO)에게 추가 여부 확인
4. 종료 또는 추가 라운드
```
